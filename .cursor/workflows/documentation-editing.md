# ドキュメント編集ワークフロー

## 対話開始前の準備

### ✅ 開いておくファイル
- [ ] 編集対象のMarkdownファイル
- [ ] 関連する INDEX.md（あれば）
- [ ] 参考にする類似ドキュメント（あれば）

### ✅ 閉じておくファイル
- [ ] スクリプト類
- [ ] テストファイル

## 新規チャット開始

**推奨**: ドキュメント編集専用の新しいチャットを開始

**理由**: コード実装のコンテキストが混入しない

---

## 効率的なクエリパターン

### パターン1: 階層ルール準拠
```markdown
@.cursor/contexts/HIERARCHY_RULES.md

{ドキュメント内容の説明}

上記のドキュメントを適切な場所に配置してください。
```

### パターン2: 命名規則準拠
```markdown
@.cursor/contexts/NAMING_RULES.md

{ファイル内容の説明}

適切なファイル名を提案してください。
```

### パターン3: 構造定義参照
```markdown
@docs/governance/DOCUMENTATION_STRUCTURE.yml

新しいセクションを追加する場合、どのTierに配置すべきですか？
```

---

## ❌ 避けるべきパターン

### 非効率なパターン1: 不要なコンテキスト
```markdown
❌ @.cursor/contexts/PERSONA_SYSTEM.md  # ドキュメント編集には不要
❌ @.cursor/contexts/PSYCHOLOGY_BACKGROUND.md  # ドキュメント編集には不要

ドキュメントを編集してください
```

---

## 編集後のチェックリスト

- [ ] **配置**: 適切なディレクトリに配置されている
- [ ] **命名**: 命名規則に準拠している
- [ ] **階層**: DOCUMENTATION_STRUCTURE.yml に準拠している
- [ ] **INDEX.md**: Git Hooks で自動生成される（手動編集不要）
- [ ] **Git add**: `git add` のみ実行（コミットは承認後）
- [ ] **ユーザー確認**: 「コミットしてもよろしいですか？」と確認
